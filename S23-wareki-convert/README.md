# S23 西暦和暦変換 — Claude Code の手足

西暦↔和暦（明治〜令和）を**決定論的**に変換するスクリプト。Claude Code に「登録」して使う手足。

## なぜ手足にするのか（＝この手足の存在理由）

LLM に元号変換を"考えさせる"と、次のような**気づきにくい間違い**を起こす：

- 元号の**境界日**：昭和64年は 1989-01-07 まで、平成元年は 1989-01-08 から。ここを1日ずらす。
- **「元年」**の扱い：令和元年＝2019、令和6年＝2024（＋5であって＋6ではない）。
- 改元年（1912 / 1926 / 1989 / 2019）が**2つの元号にまたがる**ことを見落とす。

このスクリプトに固定すれば、**同じ入力→同じ手順→同じ結果**。トークンも消費しない。
Claude Code は答えを推測せず、必ずこのスクリプトの出力（特に `--json`）を使う。

## 依存

なし。Python 3.8+ 標準ライブラリのみ（配布先でも `pip install` 不要）。

## インストール（利用者の環境に登録する）

1. `wareki.py` を手足を置くフォルダにコピーする（例：`~/.claude/hands/S23-wareki-convert/`）。
2. 下の「CLAUDE.md ルール」をプロジェクトの `CLAUDE.md` に追記する（＝発火条件の固定）。

## 使い方（CLI）

```bash
python wareki.py 2024-05-01        # → 令和6年5月1日
python wareki.py 令和6年5月1日       # → 2024-05-01
python wareki.py 昭和64             # → 1989（注記付き）
python wareki.py 1989              # → 平成元年（改元年の注記付き）
python wareki.py R6 --json         # 機械可読JSON（Claude Code はこれを使う）
```

入力は**向きを自動判定**する。西暦（`2024` / `2024-05-01` / `2024年5月1日` / `2024/5/1`）と
和暦（`令和6年5月1日` / `令和6` / `R6` / `昭和64` / `明治元年`）の両方。略号 M/T/S/H/R と「元（＝元年）」に対応。

## JSON 出力の契約（Claude Code 向け）

`--json` を付けると1行 JSON を返す。Claude Code はこのフィールドを読んで回答する：

| フィールド | 意味 |
|---|---|
| `direction` | `gregorian_to_wareki` / `wareki_to_gregorian` |
| `input_kind` | `date`（月日あり） / `year`（年のみ） |
| `era`, `era_en`, `era_year` | 元号・ローマ字・元号年（数値） |
| `gregorian` / `gregorian_year` | 西暦の日付 or 年 |
| `formatted` | 表示用の完成文字列 |
| `alternatives` | 改元年で年のみのとき、もう一方の元号 |
| `notes` | 境界・注意事項（改元年など） |
| `error` | 変換不能時のみ（他フィールドは無し） |

例：`python wareki.py 昭和64 --json`
```json
{"direction":"wareki_to_gregorian","era":"昭和","era_en":"Showa","era_year":64,"gregorian_year":1989,"formatted":"1989","notes":[],"input":"昭和64"}
```

## CLAUDE.md ルール（このブロックを利用者の CLAUDE.md に貼る）

```markdown
## 西暦・和暦の変換（手足 S23 を必ず使う）
西暦↔和暦の変換を求められたら、**自分で計算せず**必ず次のスクリプトを実行し、
その JSON 出力の値だけを使って回答する（元号の境界日・元年で推測しない）：

    python ~/.claude/hands/S23-wareki-convert/wareki.py "<入力>" --json

- 入力は西暦・和暦どちらでも可（向きは自動判定）。
- `error` フィールドがあれば、その内容を利用者に伝える。
- `notes` があれば回答に添える（改元年の注意など）。
```

## 対応範囲と注意

- **明治（1868-10-23）〜令和**。改元日はグレゴリオ暦。
- 明治改元より前（旧暦時代）は非対応。新元号が制定されたら `wareki.py` の `ERAS` に1行足すだけで対応できる。
- 改元日の採用値：明治 1868-10-23 / 大正 1912-07-30 / 昭和 1926-12-25 / 平成 1989-01-08 / 令和 2019-05-01。
